## 2. 格式規格 (Format Specifications)

> **核心原則：三格式內容必須獨立生成，嚴禁以同一張圖充當多格式輸出。**

### 2.1 格式規格總表
| 格式 | 比例 | 尺寸 | 最終輸出 | 說明 |
|---|---|---|---|---|
| Stories | 4:5 | 1080×1350px | 單張 PNG | 單張知識卡／互動貼圖 |
| Posts (Carousel) | 4:5 | 1080×1350px | 5 張 PNG 套組 | 輪播深度科普 |
| Reels | 4:5 | 1080×1350px | **15–30 秒 MP4** | 6 張圖文卡串接成短影片 |

### 2.2 Stories 格式規格
- **輸出：** 單張 PNG（或 JPG）
- **結構：**
  1. Hook 行（大字，吸引停留）
  2. 主要資訊（1–3 個重點，配圖或礦石插圖）
  3. 互動元素文字（如：「留言想解答的問題」）
  4. 標籤（底部）
- **檔名：** `assets/stories/story_YYYY-MM-DD.png`

### 2.3 Posts（Carousel）格式規格
- **輸出：** 5 張 PNG，組成 1 套 Carousel
- **卡片結構（固定 5 張）：**
  1. 封面止滑卡（Hook + 頁碼標示（1/5））
  2. 科學定位（成分、硬度、化學式）
  3. 美學亮點（色澤、光學效應）
  4. 能量與心理（脈輪、使用建議）
  5. 保養提醒 + CTA
- **檔名：** `assets/posts/post_YYYY-MM-DD_s1.png` … `_s5.png`

### 2.4 Reels 格式規格（⚠️ 最終輸出為 MP4）
- **最終輸出：** 1 支 15–30 秒 MP4 短影片
- **生成流程：**
  1. 先生成 **6 張 4:5 靜態圖文卡**（PNG）
  2. 再將 6 張圖文卡串接，輸出成 **1 支 MP4**
- **6 張卡片結構（固定）：**
  1. Hook（大字止滑，製造好奇）
  2. 知識點 A（科學定位）
  3. 知識點 B（美學亮點）
  4. 知識點 C（能量主題）
  5. 知識點 D（延伸應用／保養）
  6. CTA（點擊 / 追蹤）
- **MP4 規格：**
  - 每張停留：2.5–5 秒（總長 15–30 秒）
  - 轉場：簡潔（淡入淡出或位移），不可花巧
  - 可加輕微動態（縮放、淡入淡出），核心仍是圖文卡
  - 底部標籤全程可見
  - 禁止底部字幕；可使用畫面內圖形文字（graphic text）
- **檔名：**
  - 中間素材：`assets/reels/reel_YYYY-MM-DD_s1.png` … `_s6.png`
  - 最終影片：`assets/reels/reel_YYYY-MM-DD.mp4`
- **asset_url 回填：** 填入最終 MP4 的 URL

## 3. 內容策略 (Content Strategy)

### 3.1 三階段主題框架
| 階段 | 日期範圍 | 主題策略 | 核心 KPI |
|---|---|---|---|
| 破圈引流期 | 2026-06-15 – 2026-07-14 | 大眾痛點共鳴、礦石美學、玄學入門 | 觸及率、追蹤數 |
| 信任建立期 | 2026-07-15 – 2026-08-13 | 科普鑑定、保養禁忌、科學 vs 身心靈 | 轉發率、追蹤率 |
| 產品轉化期 | 2026-08-14 – 2026-09-12 | 生活場景、能量搭配、開運佈局 | 導購點擊率、Profile 訪問 |

### 3.2 演算法優先權（2026 對齊）
**分享 (Share) > 收藏 (Save) > 評論 (Comment) > 點讚 (Like)**

- **Shares 已超越 Likes，成為 2026 年 IG 最重要分發訊號**，Caption 與封面都必須具備「值得轉發」的角度
- **Carousel 為知識型帳號核心格式**，平均觸及與收藏表現均顯著高於單圖，應優先作為 Feed 主力輸出
- **收藏率基準：saves-to-likes ratio 目標 > 0.15**；若達標，屬高價值內容訊號
- **發布後首 90 分鐘尤其關鍵**，頭 1 小時須主動回覆留言、促進收藏與分享，加速分發
- **Caption 第一行必須是 SEO 關鍵字**，標題與首句關鍵詞覆蓋優先於堆疊 hashtag
- **Hashtag 採 3–5 個精準標籤**，避免舊式 30 個泛標籤稀釋訊號
- 封面第一行文字需兼具數字／問題／情緒觸點，以提高止滑率與翻頁率
- 頁碼標示（X/5）驅動輪播翻頁率

### 3.3 文案三層結構（科學 → 美學 → 能量）
1. **科學定位**：礦石名稱、分類、莫氏硬度、化學式、日常佩戴特性
2. **美學亮點**：色澤、天然特徵、光學效應、每顆的獨特紋理
3. **能量與心理**：對應脈輪、能量主題、適用情境、使用建議
4. **保養提醒**：針對該礦石的具體避水／避曬警告及淨化方式

## 4. 礦物學資料庫 (Mineral Database)

> ⚠️ **唯一真值源聲明**
>
> **礦石數據以 `mineralogy_data.json` 為唯一真值源。**
> 本節不重複列出礦石數據，所有圖文生成、Reels 腳本、客戶諮詢均須直接讀取 `mineralogy_data.json`。
> 缺乏資料時拒絕捏造，留空或標記「待補充」。

### 4.1 mineralogy_data.json 欄位說明
| 欄位 | 說明 |
|---|---|
| `id` | 礦石唯一識別碼 |
| `name_zh` / `name_en` | 中英文名稱 |
| `category` | 礦物分類 |
| `chemical_formula` | 化學式 |
| `mohs_hardness` | 莫氏硬度 |
| `common_colors` | 常見顏色 |
| `optical_effects` | 光學效應 |
| `natural_features` | 天然特徵（鑑別真品用） |
| `synthetic_signs` | 人工仿品特徵 |
| `chakra_primary` / `chakra_secondary` | 主副脈輪 |
| `body_focus` | 身體對應 |
| `mind_theme` | 能量主題 |
| `use_cases` | 適用情境 |
| `cleansing_ok` | 可用淨化方式 |
| `avoid_sun` / `avoid_water` | 禁忌（布爾值） |
| `notes` | 補充說明 |

### 4.2 淨化方式通則
| 方式 | 適用 | 禁用 |
|---|---|---|
| 白水晶簇淨化 | 全部 | — |
| 月光照射 | 全部 | — |
| 薰香／煙霧 | 全部 | — |
| 頌缽／音叉 | 全部 | — |
| 日光曝曬 | 白水晶（短暫） | 紫水晶、粉晶、黃水晶、月光石、蛋白石 |
| 水洗 | 硬度≥7 且無多孔結構 | 青金石、天青石、蛋白石、孔雀石、黃鐵礦 |

### 4.3 真偽鑑定準則
- **天然特徵**：冰裂紋、雲霧、色帶、不規則內含物
- **人工特徵**：氣泡、顏色過於均勻、螢光感、過輕／過重
- **最終依據**：專業寶石鑑定證書

3. **一致性**：所有協同內容保持品牌視覺規範與防幻覺協議
